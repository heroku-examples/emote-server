Event Driven Architectures

Event-driven architecture (EDA), also known as message-driven architecture, is a software architecture pattern promoting the production, detection, consumption of, and reaction to events.

Source: Wikipedia

Overview

Events ➡ Simple signals or messages that indicate something has happened, a state change, or a new request for action

 

EDA ➡ A way to communicate between components without tight coupling

 

Benefits ➡ EDA enables asynchronous, non-blocking interactions

 

 

Why Node.js?

JavaScript's non-blocking nature (Event loop)

Clicking a button on a website is an event. The browser listens to this event and sends a message to perform an action – this is essentially how EDA works at a small scale

const events = new EventEmitter();

events.on("emote", (data) => {
  const { emote } = data;
  sendToClient(emote);
});

  const { votes } = data;
  sendToClient(votes);
});

events.emit("emote", { "emote": "dolphin" });

import { EventEmitter } from 'node:events';
import { Kafka } from 'kafkajs';
const kafka = new Kafka({
  clientId: 'emotes-api',
  brokers: ['host1:port', 'host2:port', 'host3:port'],
});

const producer = kafka.producer();
const consumer = kafka.consumer({ 
  groupId: KAFKA_PREFIX + 'emotes' 
});
const topic = KAFKA_PREFIX + 'emotes';

const events = new EventEmitter();
await consumer.subscribe({ topic });
await consumer.run({
  eachMessage: async ({ 
    topic: _topic, 
    partition, 
    message 
  }) => {
    try {
      const { event, data } = 
            JSON.parse(message.value.toString());
      events.emit(event, data);
    } catch (error) {
    }
  },
});
await producer.connect();
